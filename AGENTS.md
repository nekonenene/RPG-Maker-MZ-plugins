# プロジェクト概要

このプロジェクトは、RPGツクールMZ向けのプラグイン開発プロジェクトです。  
AIエージェントはコードの生成、修正、および提案を行う際、以下のルールとコンテキストを厳守してください。

## ディレクトリとファイルの役割

- `docs/mz_classes.md` : RPGツクールMZのクラスリストが記載されています。ツクールのクラス構造を理解する際のリファレンスとして使用してください。
- `mz_scripts/` : ディレクトリの直下に、RPGツクールMZのコアとなるソースファイル（`rmmz_*.js` など）が配置されています。コアスクリプトを直接変更することは**絶対に禁止**します。参照のみに使用してください。
- `my_plugins/` : 開発中のオリジナルプラグインを格納するディレクトリです。新しいプラグインを作成する場合は、すべてこのディレクトリ内に配置してください。

## ファイル命名規則と作成ルール

- 新規プラグインのファイル名は `HTN_xxx.js` のフォーマットで作成すること。
- ファイルの先頭には、必ず以下の著作権とライセンスのコメントブロックを配置すること。年（YYYY）には作成した現在の年を入れること（継続して更新していく場合は作成年-最終更新年といった表記にする）。

```javascript
// --------------------------------------------------------------------------
//
// [Plugin Name]
//
// Copyright (c) YYYY hatonekoe
// This software is released under the MIT License.
// https://opensource.org/license/mit
//
// --------------------------------------------------------------------------
```

## コーディング規約 (プラグインの実装方法)

1. **即時実行関数 (IIFE)**
   - ファイル内の処理は必ず即時実行関数 `(function() { ... })();` でラップし、グローバルスコープを汚染しないようにしてください。
2. **アノテーションの記述**
   - 英語の `/*:` と 日本語の `/*:ja` の2つのヘルプ・パラメータ記述ブロックを必ず用意してください。ターゲットとしては `@target MZ` (または MV MZ) を用いること。
3. **既存メソッドの拡張**
   - ツクールの既存クラス（例: `Window_Base` 等）のメソッドを書き換える場合は、可能な限り「エイリアス」を利用すること。
   - 例: 
     ```javascript
     const _Window_Base_initialize = Window_Base.prototype.initialize;
     Window_Base.prototype.initialize = function(rect) {
         _Window_Base_initialize.call(this, rect);
         // 追加の処理
     };
     ```
4. **パラメータの受け取り**
   - プラグインパラメータを受け取る際は `PluginManager.parameters("HTN_xxx")` を使用し、必要に応じて型変換を行うこと。

## 情報収集に関する指示
特定の機能の処理の流れや変数名を調べる必要があるときは、推測でコードを書かずに `grep_search` 等を用いて必ず `mz_scripts/` 内のコアスクリプトを検索・参照してから実装を行ってください。
